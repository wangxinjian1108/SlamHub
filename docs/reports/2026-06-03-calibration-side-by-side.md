# 各 backend 在 7 段录制 × 3 副雷达上的逐项标定值对比

生成命令：`python3 scripts/eval_calib_side_by_side.py`

每个 cell 显示：
- **init (factory)**: application.yaml 里的工厂初值（不是 GT，但是先验）
- **consensus median**: 6 个 backend 估值的中位数（GT 缺位时的弱共识）
- **cross-backend std**: 6 个 backend 估值的 std（**< 0.1 m 表示标定可信**）
- 每个 backend 的完整 (x, y, z, roll, pitch, yaw) + 与中位数的偏差 (Δx, Δy, Δz)

读法：
- Δx/Δy/Δz 在 cm 级 = backend 跟其他基本一致
- 单个 backend 偏 0.5+ m = 它在这个 cell 上跑偏了（local minimum / 数据不足 / convention bug）
- 整 cell std > 0.3 m = 数据问题（运动量不足三角化），所有 backend 都不可信
